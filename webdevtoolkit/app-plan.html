Overview

Purpose

The purpose of Web Dev Toolkit is to help beginner web development students review and practice the major skills taught in WDD 131. The site will organize HTML, CSS, JavaScript, accessibility, responsive design, forms, arrays, objects, sorting, and filtering into one interactive learning platform.

Audience

The audience for this website is beginner web development students who are learning how to build valid, responsive, accessible, and interactive websites. The site is designed for students who need simple explanations, visual examples, and practice tools to help them understand class concepts.

Dynamic Elements

JavaScript will be used to create interactive learning features throughout the site. The Learning Topics page will display topic cards from an array of objects. Each object will include a topic title, category, difficulty level, description, and image.

Users will be able to search for topics, filter topics by category, and sort topics alphabetically or by difficulty level. JavaScript will also be used for DOM interaction, including updating the displayed cards, showing a “no results found” message, opening and closing navigation, and responding to user events.

The Toolkit page will include interactive examples where users can click buttons, submit a form, answer a quick question, and see feedback on the page. Conditional logic will be used to check user choices and display different messages depending on the result.

Website Pages

Page 1: index.html — Home Page

The Home page will introduce Web Dev Toolkit, explain what the site is for, highlight the purpose and audience, and guide users to the Learning Topics and Interactive Toolkit pages.

Page 2: topic.html — Learning Topics Page

The Learning Topics page will include searchable, filterable, and sortable WDD 131 topic cards. This page will demonstrate arrays, objects, DOM interaction, events, filtering, sorting, and responsive card layouts.

Page 3: toolkit.html — Interactive Toolkit Page

The Interactive Toolkit page will include practice demos, code examples, a playground, mini challenges, a knowledge check, form interaction, conditionals, and feedback messages.

Pseudocode for Your Project

When the Website Loads

  1. Load the page.
  2. Load the array of web development topics.
  3. Create topic cards from the array of objects.
  4. Display all topic cards on the Learning Topics page.
  5. Display the featured topic.

Navigation

  1. Wait for the user to click a navigation link.
  2. If Home is selected, display the Home page.
  3. If Topics is selected, display the Learning Topics page.
  4. If Toolkit is selected, display the Toolkit page.
  5. Highlight the active navigation item.

Search, Filter, and Sort

  1. Wait for the user to type in the search box or choose a dropdown option.
  2. Use filter to display matching topics.
  3. Use conditionals to check the selected category.
  4. Use sort to organize topics alphabetically or by difficulty.
  5. Redisplay the topic cards.
  6. If no topics match, display a “No Topics Found” message.

Interactive Examples

  1. Wait for the user to interact with an example.
  2. Detect button clicks or form input.
  3. Update the example using JavaScript.
  4. Display the results immediately.

Knowledge Check and Feedback Form

  1. Wait for the user to answer a question or complete a form.
  2. Check the selected answer or required input.
  3. If the answer or input is valid, display a success message.
  4. Otherwise, display a helpful error or try-again message.

Accessibility

  1. Display descriptive alt text for all images.
  2. Provide labels for all form controls.
  3. Allow keyboard navigation.
  4. Maintain sufficient color contrast throughout the website.
